📚 API Endpoints


                        
🔷 API Endpoints (Recommended for External Use)


                        
🔍 Server Information

                        
    
                            
  • GET /server_info - Get server information, model details, and available speakers
                                
      
                                    
    • Returns: Server version, model name, speaker list, capabilities
      • 
                                
    
                            
    • 
                        


                        
👥 Speaker Management

                        
    
                            
  • GET /audio_roles - List all available speaker presets
                                
      
                                    
    • Returns: {"success": true, "roles": ["speaker1", "speaker2", ...]}
      • 
                                
    
                            
    • 
                            
  • POST /add_speaker - Register a new speaker with reference audio
                                
      
                                    
    • Form data: name (string), audio_file (file upload)
      • 
                                    
    • Optional form data: enhance_voice (bool), super_resolution_voice (bool) — toggles ClearVoice MossFormer2_SE_48K and MossFormer2_SR_48K (both default to false); merge_backing_track (bool) mixes regenerated speech onto the extracted instrumental (requires enhancement); min_speech_ms/max_merge_ms override the segment-merging heuristics (set max_merge_ms to 0 to skip merging entirely); segments_json lets you supply Gemini-style JSON to skip inference.
      • 
                                    
    • Audio will be automatically trimmed to 3-15 seconds at silence points; when both toggles are enabled, enhancement runs before super-resolution
      • 
                                
    
                            
    • 
                            
  • POST /delete_speaker - Remove an existing speaker preset
                                
      
                                    
    • Form data: name (string)
      • 
                                
    
                            
    • 
                        


                        
🌐 Speech Translation

                        
    
                            
  • POST /api/translate_audio - Translate speech audio and regenerate voice in the target language while preserving timing.
                                
      
                                    
    • Multipart form fields: audio_file (file), dest_language (string); audio outputs are always MP3. Optional fields: prompt (custom Gemini instructions), enhance_voice (bool), super_resolution_voice (bool) to run ClearVoice preprocessing, merge_backing_track (bool) to blend the result with the extracted instrumental backing, custom_backing_audio_file (file) to override the instrumental even without enhancement, plus min_speech_ms/max_merge_ms to override segment-merging heuristics (set max_merge_ms to 0 to keep Gemini's original segmentation) and segments_json to supply pre-generated Gemini-like segments.
      • 
                                    
    • JSON alternative: {"audio": "<base64>", "dest_language": "English", "audio_mime_type": "audio/wav", "custom_backing_audio": "<base64 or url>", "custom_backing_audio_mime_type": "audio/mp3", "response_format": "mp3", "enhance_voice": true, "super_resolution_voice": false, "merge_backing_track": true, "segments_json": "[...]", "min_speech_ms": 3000, "max_merge_ms": 0} (response_format is always mp3).
      • 
                                    
    • Response: Audio stream. Inspect headers like X-Translation-Model and X-Translation-Segments for run metadata.
      • 
                                
    
                            
    • 
                        


                        
🎙️ Speech Generation - Non-Streaming (Standard Mode)

                        
    
                            
  • POST /speak - Generate speech using a registered speaker preset
                                
      
                                    
    • Required: text (string), name (speaker name)
      • 
                                    
    • Optional: response_format (always MP3 for compatibility)
      • 
                                    
    • Optional: emotion_text (emotion description), emotion_weight (0.0-1.0)
      • 
                                    
    • Optional: diffusion_steps (int, default: 10)
      • 
                                    
    • Optional: max_text_tokens_per_sentence (int, 80-200, default: 120) - Controls text splitting
      • 
                                    
    • Returns: Audio file in specified format
      • 
                                
    
                            
    • 
                            
  • POST /clone_voice - Clone voice using uploaded reference audio (zero-shot)
                                
      
                                    
    • Required: text (string), reference_audio_file (file upload)
      • 
                                    
    • Optional: Same as /speak (response_format, emotion_text, emotion_weight, diffusion_steps, max_text_tokens_per_sentence)
      • 
                                    
    • Returns: Audio file cloned from reference voice
      • 
                                
    
                            
    • 
                        


                        
⚡ Speech Generation - Streaming (Low Latency Mode)

                        
    
                            
  • POST /speak_stream - Generate speech with streaming chunks
                                
      
                                    
    • Same parameters as /speak
      • 
                                    
    • Streams audio chunks as they're generated
      • 
                                    
    • Response format: CHUNK:{idx}:{size}:{status}\n{audio_bytes}
      • 
                                    
    • Status: CONTINUE (more chunks coming) or LAST (final chunk)
      • 
                                
    
                            
    • 
                            
  • POST /clone_voice_stream - Clone voice with streaming
                                
      
                                    
    • Same parameters as /clone_voice
      • 
                                    
    • Same streaming format as /speak_stream
      • 
                                
    
                            
    •

    • JSON body: {"text": "...", "language": "auto"}
      • 
                                    
    • Returns: Estimated duration in seconds and milliseconds
      • 
                                
    
                            
    • 
                            
  • POST /api/clear_outputs - Clear all generated output files
                                
      
                                    
    • No parameters required
      • 
                                    
    • Returns: Number of files deleted and disk space freed
      • 
                                
    
                            
    • 
                        


                        


                        
🆕 Emotion Text Control Feature

                        
    
                            
  • emotion_text (optional): Natural language emotion description
                                
      
                                    
    • Examples: "happy and excited", "sad and melancholic", "calm and peaceful", "angry and frustrated"
      • 
                                
    
                            
    • 
                            
  • emotion_weight (optional): Control emotion intensity (0.0 = no emotion, 1.0 = maximum)
                                
      
                                    
    • Default: 0.6 (moderate intensity)
      • 
                                    
    • Recommended range: 0.3-0.9
      • 
                                
    
                            
    • 
                            
  • Example usage:
                                
    {
  "text": "Hello, how are you today?",
  "name": "speaker1",
  "emotion_text": "cheerful and friendly",
  "emotion_weight": 0.7,
  "response_format": "mp3"
}
    
                            
    • 
                        


                        


                        
✂️ Text Splitting Control

                        
    
                            
  • max_text_tokens_per_sentence (optional): Maximum tokens per sentence for text splitting (80-200)
                                
      
                                    
    • Default: 120 tokens
      • 
                                    
    • Range: 80-200 tokens
      • 
                                    
    • Lower values (80-100): More sentences, faster processing, may sound choppy
      • 
                                    
    • Balanced (110-130): Good balance of quality and processing speed
      • 
                                    
    • Higher values (140-200): Fewer sentences, slower processing, may impact quality for very long sentences
      • 
                                
    
                            
    • 
                            
  • Example usage:
                                
    {
  "text": "This is a long text that will be split into manageable chunks for processing.",
  "name": "speaker1",
  "max_text_tokens_per_sentence": 120,
  "response_format": "mp3"
}
    
                            
    •
